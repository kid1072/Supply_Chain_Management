# OceanBase

## 1. OceanBase

OceanBase 是一款面向企业级应用的**分布式关系型数据库**。它将数据分布在多个节点上存储和处理，通过副本、分区和一致性协议提升系统的高可用性、扩展能力与容错能力。

与单机数据库相比，OceanBase 更适合需要处理大量业务数据、较高并发访问以及持续稳定运行的场景。它支持标准 SQL，并提供 MySQL 兼容模式和 Oracle 兼容模式，便于已有业务系统进行迁移或接入。

## 2. 核心特点

### 分布式架构

OceanBase 可以将一个数据库拆分为多个分区，并将这些分区分布到不同服务器节点中。随着数据量或访问压力增加，可以通过增加节点来扩展存储和计算能力。

### 高可用与容错

OceanBase 通常通过多副本机制保存关键数据。当部分节点发生故障时，系统可以从其他可用副本中恢复或继续提供服务，从而降低单点故障对业务的影响。

### 事务与一致性

OceanBase 支持关系型数据库常用的事务能力，例如事务提交、回滚和并发控制。对于订单、库存、供应链、支付等需要保证数据一致性的业务场景，这些能力尤为重要。

### SQL 兼容性

OceanBase 支持 SQL 查询、表、索引、视图、事务等关系型数据库常见功能，并提供：

- **MySQL 模式**：适合原本使用 MySQL 的应用迁移或开发；
- **Oracle 模式**：适合需要兼容 Oracle 数据库语法和生态的业务系统。

### 横向扩展能力

当业务数据量增长或并发请求增加时，OceanBase 可以通过增加服务器节点扩展系统能力，而不必只依赖升级单台服务器的硬件配置。

## 3.在本项目中的作用

在供应链管理系统中，OceanBase 可以承担以下职责：
1. 在多用户同时访问系统时，尽量保证数据的一致性和稳定性；
2. 存储供应商、商品、仓库、库存和订单等基础业务数据；
3. 支持后端接口进行新增、查询、修改和删除操作；
4. 支持库存预警、订单分析、供应商评估等业务查询；
