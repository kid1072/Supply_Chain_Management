# 手动数据操作示例

基础数据新增和主要业务流转可在前端页面操作；修改、删除和批量维护可通过 API 文档页面操作：

```text
http://127.0.0.1:8000/docs
```

## 1. 新增商品

接口：

```text
POST /api/products
```

请求体：

```json
{
  "product_code": "DEMO-PRODUCT-001",
  "name": "Demo演示商品",
  "barcode": "690000000001",
  "category_id": 1,
  "spec": "500ml",
  "unit": "piece",
  "shelf_life_days": 365,
  "default_safety_stock": 20,
  "is_active": true
}
```

## 2. 修改商品

接口：

```text
PUT /api/products/{product_id}
```

将 `{product_id}` 替换为新增商品返回的 `id`。示例请求体：

```json
{
  "name": "Demo演示商品-已修改",
  "spec": "600ml",
  "default_safety_stock": 30,
  "is_active": true
}
```

## 3. 删除商品

接口：

```text
DELETE /api/products/{product_id}
```

将 `{product_id}` 替换为要删除商品的 `id`。如果该商品已被库存、单据或流水引用，系统可能拒绝物理删除；这种情况下可改为把商品状态设为停用。

## 4. 调整仓库库存

接口：

```text
POST /api/inventory/adjust
```

请求体：

```json
{
  "product_id": 1,
  "location_type": "warehouse",
  "warehouse_id": 1,
  "store_id": null,
  "new_quantity": 200,
  "operator_id": 1,
  "remark": "手动调整仓库库存"
}
```

## 5. 新增门店补货申请

接口：

```text
POST /api/replenishment-requests
```

请求体：

```json
{
  "store_id": 1,
  "product_id": 1,
  "request_quantity": 30,
  "request_reason": "门店库存不足，需要补货",
  "created_by": 1
}
```

## 6. 审核补货申请

接口：

```text
POST /api/replenishment-requests/{request_id}/approve?audited_by=1
```

将 `{request_id}` 替换为新增补货申请返回的 `id`。

## 7. 采用或拒绝 AI 补货建议

采用建议：

```text
POST /api/recommendations/{recommendation_id}/accept
```

拒绝建议：

```text
POST /api/recommendations/{recommendation_id}/reject
```

将 `{recommendation_id}` 替换为补货建议记录的 `id`。
