# 后端结构与运行逻辑说明

本文档用于说明当前后端仓库中各目录、主要文件的作用，以及整个后端从启动到处理请求的运行逻辑。

## 1. 项目整体技术栈

- Web 框架：FastAPI
- 数据访问层：SQLAlchemy
- 主数据库目标：OceanBase（通过 MySQL 兼容协议接入）
- 本地保底数据库：SQLite
- 测试框架：Pytest
- HTTP 启动方式：Uvicorn

`FastAPI -> Router -> Service -> SQLAlchemy Session/Model -> OceanBase 或 SQLite`

## 2. 顶层目录作用

- `app/`：后端核心代码，包含接口、配置、数据库、模型、业务逻辑、工具函数。
- `scripts/`：数据库初始化、示例数据生成与导入、重置数据的脚本。
- `tests/`：自动化测试。
- `docs/`：项目文档。
- `example/`：示例 JSON 数据。
- `schema/`：SQLite 数据库文件，以及导出的建表 SQL / 种子 SQL。
- `.env.example`：环境变量示例。
- `requirements.txt`：依赖清单。

### 3.3 core 层

- `app/core/config.py`：读取环境变量，生成全局 `Settings` 配置对象；定义主数据库 `DATABASE_URL`、SQLite 回退地址、LLM 配置、示例数据目录等。
- `app/core/database.py`：数据库连接中心。负责创建 SQLAlchemy `engine`、`SessionLocal`，优先尝试 OceanBase，失败时自动回退 SQLite，并提供当前运行数据库画像。
- `app/core/cache.py`：简单内存缓存，用于分析、库存、推荐、分布式等结果缓存。
- `app/core/response.py`：统一响应格式，如 `success_response()`、`error_response()`、`page_response()`。
- `app/core/exceptions.py`：项目内业务异常 `BusinessException`。

## 10. 整个后端的运行逻辑

### 10.1 启动阶段

```bash
uvicorn app.main:app --reload
```

启动时：

1. Python 导入 `app.main`
2. `app.main` 调用 `get_settings()` 读取配置
3. `app.main` 导入各个路由模块
4. 路由模块依赖 `app.core.database`
5. `app.core.database` 在导入阶段创建数据库引擎
6. 数据库引擎会优先尝试连接 OceanBase
7. 如果 OceanBase 不可用，则自动回退到 SQLite
8. FastAPI 应用完成初始化并注册全部接口

### 10.2 请求处理阶段

请求的处理链路如下：

1. 客户端请求某个接口，例如 `/api/products`
2. 对应 router 接收请求并通过 `Depends(get_db_dep)` 获取数据库会话
3. router 调用对应的 service 函数
4. service 通过 SQLAlchemy 查询或更新模型
5. SQLAlchemy 把操作发送到当前实际运行的数据库
6. router 使用 `success_response()` 返回统一格式
7. 若业务校验失败，抛出 `BusinessException`
8. `app.main` 中的异常处理器把异常转换成统一错误响应

### 10.3 数据库运行逻辑

当前数据库运行逻辑是：

- 首选数据库：OceanBase
- 当前环境不可连通时：自动回退 SQLite
- 当前实际运行状态可通过 `/api/health/db` 查看

返回字段里会说明：

- 当前实际方言 `dialect`
- 当前实际数据库 URL（脱敏）
- 首选主数据库 URL（脱敏）
- 当前运行模式：`oceanbase-primary` 或 `sqlite-fallback`

### 10.4 分布式模块逻辑

分布式相关能力集中在：

- `app/models/distributed.py`
- `app/services/distributed_service.py`
- `app/api/routers/distributed.py`

核心功能包括：

- 分布式库存概览
- 分布式对账
- 跨仓调拨
- 同步日志记录

项目语义：

- 分布式核心优先围绕 OceanBase 运行
- 若 OceanBase 当前不可用，则保底在 SQLite 上继续演示
- 分布式接口会把当前数据库后端信息一起返回

## 11. 后端启动代码说明

### 11.1 主启动代码

主启动入口在 `app/main.py`，核心代码逻辑如下：

```python
settings = get_settings()
app = FastAPI(title=settings.app_name)
```

这一步完成应用对象创建。

然后注册异常处理器：

```python
@app.exception_handler(BusinessException)
async def business_exception_handler(...):
    return JSONResponse(...)
```

接着把所有路由挂到应用上：

```python
for router in [
    health.router,
    users.router,
    categories.router,
    ...
]:
    app.include_router(router)
```

最后提供根接口：

```python
@app.get("/")
def root():
    return {"app": settings.app_name, "docs": "/docs", "ui": "/ui/"}
```

### 11.2 数据库启动代码

数据库启动核心在 `app/core/database.py`：

```python
engine, active_database_url, using_sqlite_fallback = _resolve_runtime_engine()
SessionLocal = sessionmaker(bind=engine, autoflush=False, autocommit=False, future=True)
```

含义是：

- 先解析首选数据库
- 优先尝试 OceanBase
- 如果 OceanBase 不通，则切换到 SQLite
- 再基于当前实际引擎创建 `SessionLocal`

### 11.3 初始化数据库代码

数据库初始化入口是：

```bash
python scripts/init_db.py --rebuild
```

逻辑是：

- 判断当前实际运行的数据库
- 如果是 SQLite，可删除本地 `.db` 后重建
- 如果是 OceanBase，直接走 `drop_all/create_all`
- 初始化默认用户

### 11.4 导入示例数据代码

命令：

```bash
python scripts/load_example_data.py
```

逻辑是：

- 若示例 JSON 不存在，先调用 `generate_example_data()`
- 再调用 `load_example_data(session)` 导入数据库

## 13. 一句话总结

这个后端的结构可以概括为：

- `app/main.py` 负责启动应用
- `router` 负责接请求
- `service` 负责业务逻辑
- `model` 负责数据库映射
- `schema` 负责请求响应结构
- `core/database.py` 负责数据库连接和 OceanBase/SQLite 运行切换
- `scripts/` 负责建库、导数和演示数据维护
